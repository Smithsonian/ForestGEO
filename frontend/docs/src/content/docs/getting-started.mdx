Welcome to the ForestGEO Data Processing Application!

## Introduction

The goal of this application is to allow you to record and analyze past and current census data for established global sites worldwide. 
By either directly entering data or uploading CSV/TSV files (conforming to defined constraints), you can save historical data and run validations or analysis on it as you complete a census.

## Logging In

Before you can log in, you must complete the following (if you haven't already):

- You must create a **personal** Microsoft (non-SI) account (if you don't have one already)
- An administrator must invite you to the Smithsonian Research Computing tenant.
- From here, your account will be added to the ForestGEO app server
- An administrator must assign at least one site to you

Once these steps are finished, you should be able to login to the application and successfully use the site.

:::note
This is a temporary measure! Once the Smithsonian TRB review process completes, ForestGEO users should be able to log in with their SI credentials.
:::

## Selection System - Getting to your Data

The application employs a selection system that enables users to choose from a site, plot, and census. After doing so, they can upload, analyze, or update information as needed.

Each subsection of the selection system appears iteratively -- for example, the site selection appears first, but only after a site is selected does the plot selection appear, and only after 
the plot is selected does the census selection appear. 


### Sites Overview and Selection

After you log in, you will be directed to the primary view of the application -- a sidebar and a dashboard view. 
Depending on the sites you have been assigned, you should see some variation of the following on the dashboard view. This has been termed the Sites Overview:

![site dashboard view (no site selected)](../../assets/dashboard-no-site-selected.png)

#### The Sites Overview

The Sites Overview denotes the grid/card view of sites that you have access to. You should be able to see the following core components: 

![sites overview card example w/ notation](../../assets/site-overview-card.jpg)

Here is an explanation of each of the highlighted objects: 

1. Site name 
2. Site schema name -- this is the database name that the site has been assigned 
3. Subquadrat size -- this is an **optional** property that allows you to set the size of each subsection of a quadrat
4. Entry mode -- by default, this is set to **single entry** -- that is, users only have to submit data once for it to be accepted. 
However, the entry mode can also be set to **double entry**, wherein users must submit data twice for it to be accepted once.

:::note
You **currently** cannot use a site's overview card to select a site! You must use the dropdown to select a site and continue to plot selection.
:::

#### The Sites Dropdown

The sites dropdown should be used to select sites. This is currently the **only** way to continue to the plot selection mechanism!

![site dropdown view](../../assets/site-dropdown-view.png)

Selecting this will change the dashboard view and open the plot selection view.

### Plot Overview and Selection

Similar to how the site overview and selection mechanism is structured, the implementation of the plots system is much the same way, save some additional editing options.

#### The Plots Overview 

Similar to the sites overview, you should see the plots present in a site as a collection of cards in a grid system. However, there are some crucial additions from the sites overview:

##### Quadrats Separation

The plots overview is divided into two primary categories -- having quadrats and not having quadrats. Plots will be arranged in alphabetical order in both categories:

![plots overview -- w/ and w/o quadrats](../../assets/plot-overview-panama.png)

The overview card for a plot is subdivided into the following core properties:

![cocoli overview card](../../assets/cocoli-plot-overview.png)

1. Number of quadrats recorded in the plot
2. Name of plot 
3. Location (including country) of the plot -- **optional**
4. Grid evaluation -- this is a rough designation of the size of your plot based on the number of quadrats it contains. It is purely for informational purposes and 
does not factor into **any** analytics or other calculations.
5. Plot editing icon button -- see the subheader for more information.

##### Plot Editing 

One of the primary differences between the sites overview card and the plots overview card is the **editing** option. Users can update the core information for a plot as needed by 
clicking on the Edit icon button. Clicking this will open this editing modal:

![cocoli-specific editing menu](../../assets/cocoli-editing-menu.png)

1. Plot Name & Details - denotes the name and general cosmetic information about the plot's location.
2. Starting Coordinates - denotes the **global** coordinates that the plot's **upper left hand corner** resides at. This should **not** be edited unless absolutely necessary -- this is a core 
part of the various validations used to determine if a stem is in the right location or not, and thus affects a large amount of core data. 
3. Plot dimensions - denotes the length x width values for the plot. This assumes that a plot is roughly rectangular, and unless this is definitively not the case, 
should be accepted as the default shape of the plot. 
4. Plot default units - denotes the baseline unit values used for each core measurement in the plot. Similar to the dimensions and coordinates, this should only be edited carefully and 
when needed.

### Creating or Selecting

This app operates along the following **core concepts**:

1. **Site**: A site is a collection of plots.
2. **Plot**: A plot is a geographic region marked for data collection
3. **Quadrat**: A subdivision of a plot -- this tends to be a standardized 20 x 20 box
4. **Census**: A census is a **date range** denoting a distinct time period where data was collected in the **Plot**

With this in mind, you will need to create a census before you can begin entering data. After opening the Census dropdown, click on the **Add New Census** button to automatically add a new census:

![add new census](../../assets/add-new-census.png)

Creating a new census will also trigger a selection reset. Please re-select your prior selections, and you should be able to see your new census displayed.

#### Deleting a Census?

If your permissions allow it, you may be able to highlight and click on a **Trashcan** icon along the **latest** census. Clicking on this will delete the census in question.

:::caution
You won't be able to delete a census if it's NOT the latest census!
:::
