## 1 · Set up site → plot → census

<ConceptCard
  title="The selection hierarchy"

### Pick (or check) your site

After login, the dashboard shows every site you've been granted access to. Use the **sidebar dropdown** to select one — the card view is informational only.

<Screenshot src={dashboardSites} alt="Sites overview cards on the dashboard" caption="Sites overview — each card shows schema name, entry mode, and subquadrat size." />

<Screenshot src={siteDropdown} alt="Site dropdown in the sidebar" width={420} caption="Selecting a site unlocks the plot view." />

:::note
Don't see your site? An administrator needs to assign it to you. Submit a help ticket from the **?** button in the lower right.
:::

### Pick or create a plot

The plots view splits plots **with quadrats** from those **without**. Click the pencil icon on a card to edit name, coordinates, dimensions, and default units.

<Screenshot src={plotsOverview} alt="Plot overview separated by quadrats" caption="Plots are alphabetised within each group." />

<Screenshot src={plotEdit} alt="Plot editing modal" width={560} caption="The plot edit modal — coordinates and units feed validations, so change them only when needed." />

### Add a census

You can add a census two ways: open the **Census** dropdown in the sidebar and click **Add New Census**, **or** click the **Start new census** action on the census overview card. Either path creates the same record — a census is just a date range you'll attach measurements to next.

<Screenshot src={addCensus} alt="Add new census action in the dropdown" width={420} caption="Adding a census resets your selection — re-pick site and plot to land on the new census." />

<Screenshot src={censusOverview} alt="Census overview with numbered actions" caption="① Delete (partial only) · ② Start a new census from the overview." />

:::caution
You can't create a new census while the current one has **zero** measurements. Import (next section) before rolling forward.
:::

## 3 · Edit the census

Open **View Data → Measurements** to see what landed. The grid is editable inline.

<Screenshot src={viewData} alt="Measurements grid with rows" caption="The measurements grid — every column is filterable; row state colours indicate validation status." />

<Screenshot src={editToolbar} alt="Edit toolbar numbered" caption="Toolbar: ① headers accordion · ② columns chooser · ③ filter · ④ clear search · ⑤ search field · ⑥ refresh · ⑦ paste · ⑧ upload · ⑨ download · ⑩ show/hide cells · ⑪ more actions." />

### Fix anything that failed

Rows that didn't pass validation appear in the **Failed Measurements** view with a structured reason. You have two ways to fix them: edit fields **in place** in the grid, or upload a CSV of corrected values as a **revision upload** — the same file format as the original import, scoped to the rows you're fixing. Either path ends with **Reingest**, which pushes the rows back through the validator.

<Screenshot src={failedMods} alt="Failed measurements modal with reasons" caption="Each failed row carries its original CSV codes plus the consolidated failure reason." />
